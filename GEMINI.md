# GEMINI.md - seadmin Project Context

## Project Overview
`seadmin` is a Rust-based TUI (Text User Interface) tool designed for analyzing and resolving SELinux access denials. It parses `ausearch` output, provides cause analysis, and offers interactive commands to apply fixes such as updating file contexts, toggling booleans, or generating custom policy modules.

### Key Technologies
- **Language:** Rust (Edition 2024)
- **UI Framework:** `ratatui` with `crossterm` backend
- **Async Runtime:** `tokio`
- **Security:** `zeroize` for secure memory handling of sudo passwords
- **I18n:** Custom macro-based system supporting 18 languages

### Architecture
- `src/main.rs`: Application entry point, main loop, and UI orchestration.
- `src/ui/`: UI components, screen management (`app.rs`), and individual screen rendering.
- `src/selinux/`: Logic for interacting with SELinux tools (`commands.rs`) and parsing AVC logs (`avc.rs`).
- `src/sudo/`: Secure execution of commands via `sudo`.
- `src/i18n/`: Internationalization definitions and detection.

## Building and Running

### Prerequisites
- Rust 1.85+
- SELinux-enabled Linux distribution (Fedora, RHEL, CentOS, etc.)
- System tools: `ausearch`, `getenforce`, `sudo`, `semanage`, `audit2allow`, `restorecon`, `setsebool`, `semodule`.

### Commands
- **Build:** `cargo build --release`
- **Run:** `cargo run` (Note: Often requires root privileges via `sudo cargo run` or running the binary with `sudo`)
- **Test:** `cargo test`
- **Lint:** `cargo clippy`
- **Format:** `cargo fmt`

## Development Conventions

### Error Handling
- Uses `anyhow::Result` for general error propagation.
- External command failures are captured and displayed to the user via a status bar.

### State Management
- The `App` struct in `src/ui/app.rs` holds the global state, including the screen stack, AVC entries, and auth context.
- UI updates are driven by an `mpsc` channel (`AppMsg`) that communicates results from background async tasks to the main UI loop.

### Internationalization
- When adding new strings, they must be added to the `Lang` trait methods in `src/i18n/mod.rs` and implemented in all language modules (e.g., `src/i18n/en.rs`, `src/i18n/ja.rs`).

### Security Practices
- Always use `Zeroizing<String>` for passwords.
- Do not log sensitive information.
- Ensure temporary files generated by `audit2allow` are cleaned up.
